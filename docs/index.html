Portable training-data SDK

Reproducible training pairs for prediction networks.

libRandomizer generates reproducible training pairs for prediction networks. Choose the input datatype, choose the output datatype, set the dataset size, and export deterministic datasets as JSON, JSONL, or CSV.

Quickstart

A training set in a few lines.

Python reference SDK

from librandomizer import TrainingDataGenerator, choice, integer

generator = TrainingDataGenerator(
    input_schema=integer(0, 99),
    output_schema=choice(["low", "medium", "high"]),
    count=100,
    seed=42,
)

pairs = generator.generate()
generator.write_jsonl("train.jsonl")
generator.write_csv("train.csv")

What ships in v1

The first implementation supports portable JSON-native input and output types: integers, numbers, booleans, strings, arrays, objects, nulls, literal values, and unions. The schema lives in data, not in an opaque runtime object.

Add a transform only when the output should be calculated from the input. Otherwise the SDK fills both sides from the selected data types.

Python/CLI is the behavior-complete reference implementation. All other language SDKs currently expose generated API surfaces and need conformance hardening before full parity claims.

Schema layer

Build nested input shapes from simple primitives.

Numbers

integer(0, 99) and number(0.0, 1.0) cover common numeric inputs.

Text

string(length=12) and choice([...]) handle tokens, labels, and categories.

Structure

array_schema(...) and object_schema(...) compose nested records.

Null and literal

null() and literal(value) keep edge cases explicit and serializable.

Two generation modes

Generate independent targets or derive them from inputs.

Schema-only pairs

generator = TrainingDataGenerator(
    input_schema=integer(0, 99),
    output_schema=choice(["low", "medium", "high"]),
    count=100,
    seed=42,
)

Use this when you need valid synthetic examples with controlled input and output datatypes.

Derived targets

generator = TrainingDataGenerator(
    input_schema=integer(0, 10),
    output_schema=number(0, 20),
    count=100,
    seed=42,
    transform=lambda value: value * 2,
)

Use this when labels, scores, or decisions should be reproducible functions of each input.

Portability

A contract language SDKs can share.

Why JSON-native schemas

Integers, floats, booleans, strings, arrays, objects, nulls, and finite choices map cleanly into the languages most developers use for data pipelines and neural network experiments.

That keeps the cross-language promise practical: the spec travels as data, while each SDK exposes native helpers and callbacks.

Legacy compatibility

The old random-value primitives are still here.

The original OS-backed random helper APIs remain available as compatibility shims while the training-data generator becomes the primary product. Existing imports keep working, but new work should build on the portable schema layer.
